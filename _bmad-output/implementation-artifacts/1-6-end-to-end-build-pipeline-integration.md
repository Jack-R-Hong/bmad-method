# Story 1.6: End-to-End Build Pipeline Integration

Status: done

## Story

As a developer,
I want a single reproducible build sequence that converts all BMAD agent `.md` files and produces a working native plugin binary,
so that the complete build process is documented and usable by any contributor from a clean checkout.

## Acceptance Criteria

**AC1:**
**Given** the `agents/` directory contains at least 3 sample `.md` agent files
**When** I run `cargo run -p bmad-converter -- --input agents/ --output crates/bmad-plugin/src/generated/` followed by `cargo build -p bmad-plugin --release`
**Then** the build completes without errors
**And** `target/release/libbmad_plugin.{so,dylib}` is produced

**AC2:**
**Given** the plugin binary exists
**When** I run `cargo test --workspace`
**Then** all workspace tests pass (unit and integration)
**And** an integration test verifies the plugin binary exports the `pulse_plugin_register` symbol (via `nm`, `objdump`, or `libloading`)

**AC3:**
**Given** the `scripts/build.sh` script exists
**When** I run `./scripts/build.sh`
**Then** it runs the converter step before the cargo build step in the correct order
**And** it exits non-zero if either step fails

**AC4:**
**Given** the `crates/bmad-plugin/src/generated/` directory
**When** I run `git status` after a build
**Then** the generated files are not tracked by git (listed in `.gitignore`)

**AC5:**
**Given** a developer clones the repository with Rust installed
**When** they follow the README build instructions step-by-step
**Then** they produce a working plugin binary without any undocumented prerequisite steps

### scripts/build.sh Content

```bash
#!/usr/bin/env bash
# BMAD-METHOD Pulse Plugin Build Script
# Usage: ./scripts/build.sh
# Runs the converter then builds the plugin binary.

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "${SCRIPT_DIR}/.." && pwd)"
AGENTS_DIR="${REPO_ROOT}/agents"
GENERATED_DIR="${REPO_ROOT}/crates/bmad-plugin/src/generated"

echo "=== BMAD-METHOD Build Pipeline ==="
echo ""

# Step 1: Run the converter
echo "Step 1/2: Running bmad-converter..."
cargo run -p bmad-converter -- \
    --input "${AGENTS_DIR}" \
    --output "${GENERATED_DIR}"
echo ""

# Step 2: Build the plugin
echo "Step 2/2: Building bmad-plugin..."
cargo build -p bmad-plugin --release
echo ""

# Report output
if [[ "$(uname)" == "Darwin" ]]; then
    PLUGIN_FILE="${REPO_ROOT}/target/release/libbmad_plugin.dylib"
else
    PLUGIN_FILE="${REPO_ROOT}/target/release/libbmad_plugin.so"
fi

if [[ -f "${PLUGIN_FILE}" ]]; then
    SIZE=$(du -sh "${PLUGIN_FILE}" | cut -f1)
    echo "=== Build complete ==="
    echo "Plugin: ${PLUGIN_FILE}"
    echo "Size:   ${SIZE}"
else
    echo "ERROR: Plugin binary not found at expected path: ${PLUGIN_FILE}" >&2
    exit 1
fi
```

### Integration Test: Symbol Export Verification

`tests/plugin_integration.rs`:

```rust
// tests/plugin_integration.rs
// Verifies the built plugin binary exports the required symbol.
// Must run AFTER `cargo build -p bmad-plugin --release`.

use std::path::Path;
use std::process::Command;

fn plugin_path() -> std::path::PathBuf {
    let manifest_dir = Path::new(env!("CARGO_MANIFEST_DIR"));
    let target = manifest_dir.join("target").join("release");
    if cfg!(target_os = "macos") {
        target.join("libbmad_plugin.dylib")
    } else {
        target.join("libbmad_plugin.so")
    }
}

#[test]
fn plugin_exports_register_symbol() {
    let path = plugin_path();
    if !path.exists() {
        eprintln!("Skipping: plugin binary not found at {:?}", path);
        eprintln!("Run `cargo build -p bmad-plugin --release` first.");
        return; // Skip gracefully — don't fail CI before first build
    }

    // Check symbol export using `nm` (available on Linux and macOS)
    let output = Command::new("nm")
        .arg("--dynamic")
        .arg("--defined-only")
        .arg(&path)
        .output();

    match output {
        Ok(out) => {
            let stdout = String::from_utf8_lossy(&out.stdout);
            assert!(
                stdout.contains("pulse_plugin_register"),
                "Plugin binary does not export 'pulse_plugin_register' symbol.\nnm output:\n{}",
                stdout
            );
        }
        Err(e) => {
            // nm not available — try objdump
            eprintln!("nm not available ({}), trying objdump...", e);
            let output2 = Command::new("objdump")
                .arg("-T")
                .arg(&path)
                .output()
                .expect("neither nm nor objdump available");
            let stdout = String::from_utf8_lossy(&output2.stdout);
            assert!(
                stdout.contains("pulse_plugin_register"),
                "Plugin binary does not export 'pulse_plugin_register' symbol.\nobjdump output:\n{}",
                stdout
            );
        }
    }
}
```

### Integration Test: Converter End-to-End

`tests/converter_integration.rs`:

```rust
// tests/converter_integration.rs
use std::io::Write;
use tempfile::TempDir;

fn write_agent(dir: &std::path::Path, name: &str) -> std::path::PathBuf {
    let path = dir.join(format!("{}.md", name));
    let content = format!(r#"---
name: {name}
displayName: "{Name} Agent"
description: "Test {name} agent"
executor: bmad/{name}
capabilities:
  - {name}-capability
---

# {Name}

You are {Name}, a test agent.
"#, name = name, Name = {
        let mut c = name.chars();
        match c.next() {
            None => String::new(),
            Some(f) => f.to_uppercase().collect::<String>() + c.as_str(),
        }
    });
    std::fs::write(&path, content).unwrap();
    path
}

#[test]
fn converter_produces_one_file_per_agent_plus_mod() {
    let input_dir = TempDir::new().unwrap();
    let output_dir = TempDir::new().unwrap();

    write_agent(input_dir.path(), "alpha");
    write_agent(input_dir.path(), "beta");

    // Run the converter binary
    let status = std::process::Command::new(env!("CARGO_BIN_EXE_bmad-converter"))
        .arg("--input")
        .arg(input_dir.path())
        .arg("--output")
        .arg(output_dir.path())
        .status()
        .expect("failed to run bmad-converter");

    assert!(status.success(), "Converter exited with non-zero status");

    // Check output files
    let mod_rs = output_dir.path().join("mod.rs");
    assert!(mod_rs.exists(), "mod.rs must exist");

    let alpha_rs = output_dir.path().join("alpha.rs");
    assert!(alpha_rs.exists(), "alpha.rs must exist");

    let beta_rs = output_dir.path().join("beta.rs");
    assert!(beta_rs.exists(), "beta.rs must exist");

    // Check headers
    let alpha_content = std::fs::read_to_string(&alpha_rs).unwrap();
    assert!(
        alpha_content.starts_with("//! Auto-generated by bmad-converter. DO NOT EDIT."),
        "Generated file must start with standard header"
    );
}

#[test]
fn converter_clean_overwrites_stale_files() {
    let input_dir = TempDir::new().unwrap();
    let output_dir = TempDir::new().unwrap();

    // Create a stale file
    let stale = output_dir.path().join("stale_agent.rs");
    std::fs::write(&stale, "// stale").unwrap();

    write_agent(input_dir.path(), "fresh");

    let status = std::process::Command::new(env!("CARGO_BIN_EXE_bmad-converter"))
        .arg("--input")
        .arg(input_dir.path())
        .arg("--output")
        .arg(output_dir.path())
        .status()
        .expect("failed to run bmad-converter");

    assert!(status.success());
    assert!(!stale.exists(), "Stale file must be removed during clean regeneration");
}
```


### Platform Binary Names

| OS | Binary name |
|----|-------------|
| Linux x86_64 | `target/release/libbmad_plugin.so` |
| Linux aarch64 | `target/release/libbmad_plugin.so` |
| macOS x86_64 | `target/release/libbmad_plugin.dylib` |
| macOS aarch64 | `target/release/libbmad_plugin.dylib` |

The binary name is derived from the crate name `bmad-plugin` → `libbmad_plugin` with the platform-specific extension.

### Test Dependency: CARGO_BIN_EXE_bmad-converter

The integration test uses `env!("CARGO_BIN_EXE_bmad-converter")` to locate the built binary. This macro is set by Cargo for integration tests and resolves to the path of the `bmad-converter` binary. This works without `CARGO_BIN_EXE_` only if the binary is built before tests run.

For CI: ensure `cargo test --workspace` is preceded by `cargo build --workspace` or the converter binary exists.
